Steg-för-steg

Sex steg från noll till en live VM i Centrera Cloud Portal.

1

Grundläggande – välj namn

Ge din virtuella server ett namn som identifierar den i portalen.

Centrera Cloud Portal
Namn: min-demo-vm
2

Operativsystem – välj OS

Välj vilket operativsystem din VM ska köra. Ubuntu LTS rekommenderas för cloud-init-kompatibilitet.

Centrera Cloud Portal
OS: Ubuntu 22.04 LTS
3

Storlek – välj resurser

Dimensionera CPU och RAM efter behov. En liten flavor räcker för en webbdemo.

Centrera Cloud Portal
Flavor: m1.small (2 vCPU / 4 GB RAM)
4

Nätverk – välj subnät

Koppla VM:en till rätt nätverk och subnät för att få en IP-adress.

Centrera Cloud Portal
Subnät: demo-network / 10.0.0.0/24
5

Anpassning – klistra in cloud-init

Klistra in cloud-init-konfigurationen. Den installerar Apache, klonar repot och anpassar sidan med kundnamn – helt automatiskt vid första start.

User Data / cloud-init
#cloud-config
packages: [apache2, git, ufw]
runcmd:
  - git clone https://github.com/...
  - cp vm/ci/index.html /var/www/html/
  - sed -i 's/KUND_NAMN/Kund X/' /var/www/html/index.html
6

Säkerhet – SSH-nyckel

Välj eller klistra in din publika SSH-nyckel. VM:en är nu redo – starta den och sidan är live inom en minut!

Centrera Cloud Portal
SSH-nyckel: ssh-rsa AAAA... din@nyckel

Varför cloud-init?

Det enklaste sättet att automatisera en VM från dag ett – utan manuell konfiguration.

⚡

Snabb uppstart

Hela miljön – paket, brandvägg, webbserver och innehåll – är redo på under en minut utan att du behöver logga in.

🔁

Repeterbar

Samma cloud-init-fil ger exakt samma resultat varje gång. Perfekt för demo-miljöer och kundpresentationer.

🎨

Kundanpassad

Byt ett enda värde i filen – KUND_NAMN – och sidan anpassas automatiskt för rätt kund.

🔒

Säker från start

UFW-brandväggen konfigureras automatiskt: port 22, 80 och 443 öppna – allt annat blockerat.
