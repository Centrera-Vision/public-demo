Steg-för-steg

Fyra kommandon från noll till en snygg webbsida i Kubernetes.

1

Skapa Deployment

Startar en nginx-pod i namespace demo.

Terminal
$ kubectl create deployment nginx --image=nginx -n demo
2

Exponera med LoadBalancer

Skapar en extern IP via molnleverantörens load balancer.

Terminal
$ kubectl expose deployment nginx \
  --type=LoadBalancer \
  --port=80 \
  --target-port=80 \
  -n demo
3

Skapa ConfigMap från index.html

Läser in din lokala index.html direkt som en Kubernetes-resurs – ingen YAML behövs.

Terminal
$ kubectl create configmap nginx-html \
  --from-file=index.html=index.html \
  -n demo
4

Patcha Deployment att mounta ConfigMap

Mountar ConfigMap:en till nginx:s HTML-katalog. Allt inline – ingen extra fil behövs.

Terminal
$ kubectl patch deployment nginx -n demo --patch ' spec: template: spec: containers: - name: nginx volumeMounts: - name: html mountPath: /usr/share/nginx/html volumes: - name: html configMap: name: nginx-html'
5

Kontrollera att det rullar ut OK

Väntar tills den nya pod:en är uppe och redo.

Terminal
$ kubectl rollout status deployment nginx -n demo
6

Hämta extern IP

När EXTERNAL-IP dyker upp är sidan live – öppna den i webbläsaren!

Terminal
$ kubectl get service nginx -n demo

Varför ConfigMap?

Det enklaste sättet att anpassa statisk webbinnehåll utan att bygga egna images.

⚡

Snabb uppdatering

Ändra innehållet med kubectl edit och en rollout restart – inga pipelines, inga registries.

🔒

Ingen extern beroende

HTML lagras direkt i klustret som en Kubernetes-resurs. Inget behov av extern fillagring.

📦

Standard nginx-image

Du använder officiel nginx-image utan modifikationer. Lättare att hålla uppdaterad och säker.

🔁

GitOps-vänligt

ConfigMap är en vanlig YAML-fil som versionshantering i Git precis som all annan infrastruktur.

Redo att bygga vidare?

Nästa steg: Ingress, TLS-certifikat och Helm-charts.
